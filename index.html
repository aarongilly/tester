Reminder for myself

I knocked this out one night after my kid went down. It's not as difficult as I feared. Here are the basic steps:

  1. VS Code
    1. npm init
    2. in package.json > scripts, supply a "start" key with value "node index.js"
    3. npm express install
    4. add a couple routes for fun
  2. GitHub
    1. new repository
    2. name it whatever
  3. Back in VS Code
    1. Source Control > Push to... > add new remote
    2. Source Control > Push to... > (newly added remote)
  4. Glitch
    1. New Project > Import from GitHub

I am and was able to debug in VS Code using the Chrome Debugging extension. Set breakpoints and everything. It worked very well for a bit.

I did run into some trouble when trying to pull from GitHub to my existing Glitch project, though. Glitch is giving me an error. Also it seems like it just fully overwrites it every time - not ideal.

I believe I can skip the GitHub step altogether and just create the new remote and point it directly at the git link in the glitch app console. Although I never got this working. I would just use the Glitch extension for that purpose.
